# Implementation Plan: SQLite Storage for Pipeline and Applications

**Branch**: `001-sqlite-storage` | **Date**: 2026-04-18 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-sqlite-storage/spec.md`

## Summary

Replace `data/applications.md` and `data/pipeline.md` as the source of truth
with a SQLite3 database at `data/career-ops.db`. Add a `db.mjs` CLI with named
subcommands (`list`, `get`, `insert`, `update`, `delete`, `content`, `export`,
`migrate`, `stats`) that returns JSON on stdout — the canonical interface for
both the AI agent and all `.mjs` scripts. Migrate every script that currently
parses markdown (`merge-tracker`, `verify-pipeline`, `dedup-tracker`,
`normalize-statuses`, `analyze-patterns`, `followup-cadence`, `check-liveness`,
`scan`, batch ingest, and the Go dashboard) to read/write the DB instead.
Retain `applications.md` and `pipeline.md` as regenerated read-only snapshots
via `db.mjs export`. Provide a `migrate-to-db.mjs` one-shot migration for
existing users.

## Technical Context

**Language/Version**: Node.js v25 (ES modules, `.mjs`) + Go (dashboard TUI)
**Primary Dependencies**: `better-sqlite3` (sync Node.js SQLite driver — new),
  `modernc.org/sqlite` (pure-Go SQLite driver — new), `js-yaml` (existing),
  `playwright` (existing)
**Storage**: SQLite3 — `data/career-ops.db` (user-layer, local file)
**Testing**: `test-all.mjs` (existing custom runner, extended with DB tests)
**Target Platform**: macOS / Linux local environment, no network dependency
**Project Type**: CLI scripts (Node.js) + Go TUI dashboard
**Performance Goals**: Single-row lookup and simple filter queries under 100 ms
  on a 10,000-row dataset; `db.mjs export` regenerates markdown snapshots in
  under 5 seconds
**Constraints**: Local-first; no server process; `data/career-ops.db` is
  User Layer (not overwritten by `update-system.mjs apply`)
**Scale/Scope**: Hundreds to low-thousands of records; low write volume
  (dozens/minute peak in batch mode); concurrent batch workers require
  SQLite WAL mode

## Complexity Tracking

> No constitution violations to justify.

| Complexity introduced | Why needed | Simpler alternative rejected because |
|-----------------------|------------|--------------------------------------|
| Two language SQLite drivers (better-sqlite3 + modernc.org/sqlite) | Dashboard is Go; Node scripts are JS. Both must read the same DB file. | Single-language rewrite of dashboard in Node.js would be a far larger scope change outside this feature. |
| WAL mode on SQLite | Concurrent batch workers (FR-016) can deadlock on default journal mode. | Default journal mode causes `SQLITE_BUSY` errors under concurrent writes; WAL enables concurrent readers + one writer without blocking. |
